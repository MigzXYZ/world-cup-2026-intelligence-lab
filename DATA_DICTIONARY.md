# Data Dictionary

## Core rule

The app must not invent missing official facts. Missing data should remain blank/NA and be shown in Data Health.

## `data/processed/teams_current.csv`

Generated by `python -m src.pipelines.update_all`.

Required columns:

| Column | Description |
|---|---|
| team | Canonical team name. |
| group | World Cup group. |
| confederation | Confederation, such as UEFA, CAF, AFC, CONMEBOL, CONCACAF, OFC. |
| is_host | 1 if host/co-host, otherwise 0. |

Recommended columns:

| Column | Description |
|---|---|
| fifa_rank | Official FIFA ranking snapshot rank. Lower is better. |
| fifa_points | Official FIFA ranking points if available. |
| elo_rating | World Football Elo or selected Elo snapshot. Higher is better. |
| recent_form | Recent-form score created from historical results. Higher is better. |
| goals_for_pg | Goals scored per game in recent sample. |
| goals_against_pg | Goals conceded per game in recent sample. Lower is better. |
| attack_index | Optional precomputed attack strength. |
| defense_index | Optional precomputed defense strength. |
| model_rating_seed | Fallback demo rating only. |
| model_rating_final | Final app rating produced by the model layer. |
| source_updated_at_utc | Timestamp created by the pipeline. |

## `data/processed/matches_current.csv`

Required columns:

| Column | Description |
|---|---|
| match_id | Unique match identifier. |
| stage | Group Stage, Round of 32, Round of 16, Quarter-final, Semi-final, Final, etc. |
| group | Group letter for group-stage matches. |
| team_a | First team. |
| team_b | Second team. |
| status | scheduled, live, finished, postponed, cancelled, delayed. |

Recommended columns:

| Column | Description |
|---|---|
| date | Match date in YYYY-MM-DD format. |
| time_local | Local match time. |
| venue | Stadium or venue name. |
| city | Host city matching `host_cities_current.csv`. |
| country | Host country. |
| score_a | Final score for team_a if finished. |
| score_b | Final score for team_b if finished. |
| winner | Winner for knockout matches if finished. |
| source_updated_at_utc | Timestamp created by the pipeline. |

## `data/processed/rankings_current.csv`

Supported wide format:

| Column | Description |
|---|---|
| team | Canonical team name. |
| fifa_rank | FIFA rank. |
| fifa_points | FIFA points if available. |
| elo_rating | Elo rating. |
| snapshot_date | Date of ranking snapshot. |

Supported long format:

| Column | Description |
|---|---|
| team | Canonical team name. |
| source | `fifa_rank`, `fifa`, or `elo`. |
| rating_value | Numeric value. |
| snapshot_date | Date of ranking snapshot. |

## `data/processed/data_health_report.csv`

Generated validation output.

| Column | Description |
|---|---|
| checked_at_utc | Validation timestamp. |
| severity | ok, warning, or error. |
| dataset | Dataset being checked. |
| check | Validation rule name. |
| message | Human-readable issue. |

## `data/source_logs/latest_source_log.csv`

Generated by the pipeline.

| Column | Description |
|---|---|
| source_name | Source category. |
| source_type | remote_csv, official_local, seed_fallback, template_fallback, etc. |
| source_ref | File path or environment variable name. |
| row_count | Number of rows loaded. |
| updated_at_utc | Pipeline timestamp. |
